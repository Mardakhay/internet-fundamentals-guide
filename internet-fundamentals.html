What is the Internet

A network is a group of computers or devices connected to each other. The internet is a network of networks — when individual home, office, and institutional networks are all connected together, they form the global internet.

The internet was developed in the late 1960s by the United States Department of Defense as a decentralized communication system. Today it serves billions of users worldwide and forms the foundation upon which every modern software application is built.

⚙️

How the Internet Works

At a high level, the internet connects devices through a set of standardized protocols that define how information is exchanged and ensure data is transmitted reliably and securely.

The core of the internet is a global network of interconnected routers, which direct traffic between devices and systems. When you send data, it is broken into small units called packets. Each packet travels through a series of routers until it reaches its destination, where the packets are reassembled into the original message.

Network Protocols

A protocol is a set of rules and standards defining how information is exchanged between devices. Protocols allow devices from different manufacturers to communicate seamlessly.

Core Protocols

IP

Internet Protocol

Routes packets to their correct destination using IP addresses. Every device on the internet is assigned an IP address used to identify and locate it.

TCP

Transmission Control Protocol

Ensures packets are delivered reliably, in the correct order, and without errors. Establishes a formal connection (via handshake) before data transfer begins.

UDP

User Datagram Protocol

A faster but less reliable alternative to TCP. Suitable for real-time applications such as video streaming and online gaming where speed is prioritized over guaranteed delivery.

DNS

Domain Name System

Translates human-readable domain names into machine-readable IP addresses.

HTTP/S

Hypertext Transfer Protocol (Secure)

Defines how content is requested and delivered between web clients and servers. HTTPS adds TLS encryption.

IP Addresses & Domain Names

An IP address is a unique numerical identifier assigned to every device on a network. It ensures data packets are routed to the correct destination.

IPv4

IPv6

Alphanumeric notation supporting an exponentially larger address space for modern devices.

Domain Names

Domain names are human-readable identifiers for websites (e.g., google.com). Because computers communicate using IP addresses, the Domain Name System (DNS) is responsible for translating domain names into IP addresses.

📚

Domain Name System (DNS)

🔖 DNS is often described as the phonebook of the internet. It translates human-readable domain names into the IP addresses machines require. Without DNS, users would need to memorize a numerical IP address for every website they visit.

The Four DNS Server Types

1

DNS Recursor

Receives queries from client applications (like a browser) and makes further requests to fully resolve the query.

2

Root Nameserver

First stop in the resolution chain. Doesn't know the final IP but directs the recursor to the appropriate TLD nameserver.

3

TLD Nameserver

Handles the last segment of a domain (e.g., .com, .org). Directs the recursor to the authoritative nameserver.

4

Authoritative Nameserver

Final stop. Holds the actual DNS record and returns the corresponding IP address to the recursor.

DNS records are cached at multiple levels to reduce lookup times. Each record is stored for a duration defined by a Time-to-Live (TTL) value set by the domain owner.

HTTP & HTTPS

HTTP (Hypertext Transfer Protocol) is a TCP/IP-based, application-layer protocol that standardizes how clients and servers communicate. By default, HTTP operates on TCP port 80.

HTTPS

HTTPS (HTTP Secure) is the encrypted version of HTTP. It uses SSL/TLS to encrypt all data between client and server, protecting credentials, session tokens, and payment data from interception. HTTPS operates on TCP port 443. Browsers indicate a secure connection with a padlock icon in the address bar.

🔒

SSL / TLS

SSL (Secure Sockets Layer) and its successor TLS (Transport Layer Security) are cryptographic protocols that provide secure, encrypted communication. SSL is deprecated — TLS is the current standard, though "SSL" is still used informally.

STEP 1 — CERTIFICATES

Certificate Verification

TLS certificates are issued by trusted Certificate Authorities (CAs). They contain the server's public key and identity info, allowing the client to verify it's communicating with the legitimate server.

STEP 2 — HANDSHAKE

TLS Handshake

Client and server negotiate the encryption algorithm, verify the certificate, and exchange keys — all before any application data is transmitted.

STEP 3 — ENCRYPTION

Encrypted Data Transfer

Once the handshake is complete, all data is encrypted using the agreed cipher. Content is unreadable to any third party intercepting the connection.

✅ Best Practice: Enforce TLS version 1.2 or higher, maintain valid certificates, and follow current configuration best practices for any application transmitting sensitive data.

Sockets & Connections

A socket is the combination of an IP address and a port number, representing a specific communication endpoint. A TCP connection is established between two sockets — one on the client, one on the server — through the three-way handshake. Common application-layer protocols built on TCP/IP include HTTP, FTP, and SMTP.

Web Hosting

Web hosting is a service that allocates storage space on a server for a website's files, making them accessible via the internet through a domain name and IP address. When a visitor enters a domain name, DNS resolves it to the host server's IP address, and the server delivers the requested files via HTTP.

Server Resources

CPU

Handles all server-side processing. Greater power enables faster page generation.

RAM

Manages data actively in use. More memory supports higher concurrent requests.

Storage

Determines how much content can be hosted. SSDs and NVMe reduce retrieval times.

Bandwidth

Volume of data transferable per unit time. Insufficient bandwidth causes slowdowns under traffic.

Web Browsers

A web browser is a software application that enables users to access, retrieve, and display content from the World Wide Web. In the client-server model, the browser is the client — it sends HTTP requests to web servers and renders returned HTML, CSS, and JavaScript into visual, interactive pages.

The major browsers today are Google Chrome, Mozilla Firefox, Apple Safari, and Microsoft Edge.

How a Browser Processes a Request

1

DNS Lookup

Browser checks its DNS cache; if no record exists, performs a full DNS lookup to resolve the domain to an IP address.

2

TCP + TLS Handshake

Browser establishes a TCP connection with the server. For HTTPS, a TLS handshake is completed before any data is sent.

3

HTTP GET Request

Browser sends an HTTP GET request for the resource.

4

Server Response

Server returns an HTTP response containing the HTML document.

5

Rendering Pipeline

Rendering engine parses HTML → builds DOM. Processes CSS → builds CSSOM. Combines both into a render tree, calculates layout, and paints the final result to screen.

6

JavaScript Execution

JS files are fetched and executed by the JavaScript engine, which can modify the DOM and CSSOM dynamically after the initial page load.

Browser Security

HTTPS

HTTPS Enforcement

Modern browsers warn users on unencrypted HTTP connections and block mixed content — HTTP resources loaded within an HTTPS page.

🏖️

Sandboxing

Each browser tab runs in an isolated process. Code in one tab cannot access the OS, other tabs, or other applications.

🔐

Privacy Controls

Block third-party cookies, manage per-site permissions, clear browsing data. Private browsing mode retains no history, cookies, or form data after the session ends.
